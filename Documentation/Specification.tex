\section{Introduction}
The goal of this project is to implement a simple 16-bit accumulator machine in the style of the 16-bit minicomputers of the early 1970s. 
This is not designed to be a fast processor, nor is it designed to be an easy compiler target. 
Rather, it is designed to be easily implemented with TTL logic, and microcode.

\subsection{The Accumulator}
The accumulator will be used to store the results of calculations. 
Especially those performed by ARI-format instructions.

\subsection{The Index Register}
This register will be used of indexing memory.
The direct addressing mode will allow compilers to use this register as a `frame pointer'.

\subsection{The Program Counter}
This register will contain the address of the next instruction to be executed. 
It will be incremented during each instruction fetch cycle. 
Writing to this register will result in a `jump.'

\subsection{The Zero Register}
This register will be fixed to zero. 
Writing to this register will have no effect on its contents.
Those who have studied MIPS-style RISC architectures will understand the utility of such a register.

\subsection{The Link Register}
This register will be used to store the overflow bit of recent additions. 
Conditional instructions will only execute if this bit is set.

\subsection{Addressing Mode, Register, Immediate (ARI)}
This instruction format will supply an addressing mode (`MODE'), register (`BASE') and immediate (`IMM').
These fields will be used to calculate an `effective address'. The contents of memory at this effective address will be supplied 
to the operation.
